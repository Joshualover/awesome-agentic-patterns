# Contributing to Awesome Agentic Patterns

Thank you for your interest in contributing to this curated list of agentic AI patterns! Your contributions help the entire AI agent community.

## How to Contribute

### Quick Links

- 🐛 [Report an issue](https://github.com/Scottcjn/awesome-agentic-patterns/issues/new)
- 💡 [Suggest a pattern](https://github.com/Scottcjn/awesome-agentic-patterns/issues/new)
- 🔀 [Submit a pull request](https://github.com/Scottcjn/awesome-agentic-patterns/compare)

### Types of Contributions

We welcome the following types of contributions:

1. **New Patterns** - Add new agentic patterns with descriptions and examples
2. **Pattern Improvements** - Expand existing patterns with more details
3. **Real-world Examples** - Add case studies or implementations
4. **References** - Add academic papers, blog posts, or tools
5. **Fixes** - Correct errors, broken links, or typos
6. **Translations** - Translate to other languages

## Contribution Guidelines

### Before Contributing

1. **Search existing patterns** - Check if the pattern already exists
2. **Check open issues/PRs** - Avoid duplicate work
3. **Ensure quality** - Patterns should be well-documented and practical

### Adding a New Pattern

When adding a new pattern, please include:

#### Required Information

1. **Pattern Name** - Clear, descriptive name
2. **Description** - 2-3 sentences explaining the pattern
3. **Use Case** - When should someone use this pattern?
4. **Example** - Concrete example or pseudo-code
5. **References** - At least one reference (paper, tool, or implementation)

#### Format

Use this template:

```markdown
### Pattern Name
- **Description**: Brief explanation of the pattern (1-2 sentences)
- **Use Case**: When and why to use this pattern
- **Example**: 
  ```python
  # Concrete example or pseudo-code
  ```
- **Reference**: 
  - [Paper Title](link) - Brief description
  - [Tool Name](link) - Brief description
  - [Blog Post](link) - Brief description
```

#### Pattern Criteria

Patterns should meet these criteria:

- [ ] **Practical** - Can be implemented and used today
- [ ] **Well-defined** - Clear scope and boundaries
- [ ] **Useful** - Solves a real problem in agentic systems
- [ ] **Documented** - Has references or implementations
- [ ] **Novel** - Not a duplicate of existing patterns

### Editing Existing Patterns

When improving existing patterns:

- Keep the original meaning intact
- Add value (examples, references, clarifications)
- Maintain consistent formatting
- Explain complex concepts simply

## Submission Process

### 1. Fork and Branch

```bash
# Fork the repo on GitHub, then:
git clone https://github.com/YOUR_USERNAME/awesome-agentic-patterns.git
cd awesome-agentic-patterns
git checkout -b add-pattern-name
```

### 2. Make Your Changes

- Edit `README.md` for pattern changes
- Add examples if helpful
- Update this file if adding new categories

### 3. Follow Style Guidelines

#### Writing Style

- Use clear, concise language
- Write in present tense
- Use active voice when possible
- Keep descriptions under 80 characters per line when possible

#### Formatting

- Use proper Markdown formatting
- Add blank lines around headers
- Use code blocks for examples
- Use bullet points for lists

#### Links

- Use descriptive link text (not "click here")
- Ensure links are accessible (no paywalls if possible)
- Prefer official sources
- Use HTTPS when available

### 4. Commit Your Changes

```bash
git add README.md
git commit -m "Add pattern: Pattern Name

- Add description and use cases
- Include Python example
- Reference research paper and tools"
```

### 5. Submit Pull Request

1. Push to your fork:
   ```bash
   git push origin add-pattern-name
   ```
2. Open a Pull Request on GitHub
3. Fill out the PR template:
   - What pattern you're adding/updating
   - Why it should be included
   - References or evidence of usage

## PR Review Process

1. **Automated checks** - CI will check formatting and links
2. **Maintainer review** - A maintainer will review within 7 days
3. **Feedback** - Address any requested changes
4. **Merge** - Once approved, your contribution will be merged

## Quality Standards

### What Makes a Good Pattern

✅ **Good patterns:**
- Solve real problems in agentic AI
- Have clear, actionable descriptions
- Include practical examples
- Are backed by research or real-world usage
- Are technology-agnostic when possible

❌ **Avoid:**
- Vendor-specific marketing
- Unproven or theoretical concepts
- Duplicate patterns
- Overly complex explanations
- Broken or paywalled links

### Pattern Categories

If your pattern doesn't fit existing categories:

1. Consider if it fits in a subcategory
2. Propose a new category in your PR description
3. Ensure the new category has at least 2-3 patterns

## Code of Conduct

### Our Standards

- Be respectful and constructive
- Welcome newcomers
- Accept constructive criticism gracefully
- Focus on what's best for the community

### Unacceptable Behavior

- Harassment or discrimination
- Trolling or insulting comments
- Personal or political attacks
- Publishing others' private information
